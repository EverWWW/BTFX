# 02 - 数据模型与存储设计

## 一、模型总览

```
新增模型
├── AnalysisResult           ── 分析结果主表（关联 MeasurementRecord）
├── KinematicSummary         ── 运动学汇总参数
├── AnalysisTaskConfig       ── 算法输入配置（对应 task_config.json）
├── AnalysisSummary          ── 算法输出汇总（对应 summary.json 反序列化）
├── TaskStatusMessage        ── 实时状态消息（stdout JSON）
├── QualityControlInfo       ── 质量控制信息
├── AnalysisRequest          ── 分析请求 DTO（ViewModel → Service）
├── AnalysisOptions          ── 分析选项
└── AlgorithmSettings        ── 算法配置（AppSettings 扩展）

扩展模型
├── GaitParameters           ── 新增运动学关联字段
└── MeasurementRecord        ── 新增分析结果关联
```

## 三、现有模型扩展

### 3.1 GaitParameters 扩展

现有 `GaitParameters` 模型的字段基本能覆盖步态事件参数，但需要增强与算法输出的映射：

| 现有字段 | 算法输出字段 | 映射说明 |
|----------|-------------|---------|
| StrideLengthLeft/Right | stride_length_m | 算法暂输出单值，待左右分别输出 |
| StepLengthLeft/Right | step_length_m | 同上 |
| Cadence | cadence_step_per_min | 直接映射 |
| Velocity | gait_speed_m_per_s | 直接映射 |
| StancePhaseLeft/Right | stance_time_s | 需换算为百分比或存储原始秒数 |
| SwingPhaseLeft/Right | swing_time_s | 同上 |
| DoubleSupport | double_support_time_s | 同上 |
| SingleSupport | single_support_time_s | 同上 |

**新增字段建议**：

```
GaitParameters 新增字段
├── AnalysisResultId (int?)       ── 关联分析结果（可选，向前兼容）
├── GaitCycleDurationS (double?)  ── 步态周期时长(s)
├── StanceTimeS (double?)         ── 站立相时间(s)（原始值）
├── SwingTimeS (double?)          ── 摆动相时间(s)（原始值）
├── DoubleSupportTimeS (double?)  ── 双支撑相时间(s)（原始值）
├── SingleSupportTimeS (double?)  ── 单支撑相时间(s)（原始值）
├── StepLengthM (double?)         ── 步长(m)（算法原始单值）
├── StrideLengthM (double?)       ── 步幅(m)（算法原始单值）
└── GaitSpeedMPerS (double?)      ── 步速(m/s)（算法原始值）
```

### 3.2 MeasurementRecord 扩展

新增分析结果导航属性：

```
MeasurementRecord 新增
├── AnalysisResults (List<AnalysisResult>?)  ── 分析结果列表（导航属性）
└── LatestAnalysisResult (AnalysisResult?)   ── 最新成功分析结果（导航属性）
```

## 六、数据库迁移策略

由于项目使用 SqlSugar ORM，新增表可通过 `DatabaseInitializer` 的 `CodeFirst` 方式自动创建：

```
DatabaseInitializer 扩展
├── 新增 AnalysisResults 表
├── 新增 KinematicSummaries 表
├── 新增 AnalysisCsvFiles 表
├── 新增 QualityControls 表
└── GaitParameters 表增加新字段（需处理已有数据兼容性）
```

### 注意事项
- GaitParameters 的新增字段全部设为 `IsNullable = true`，兼容已有数据
- 新增的 `AnalysisResultId` 外键也设为可空，因为历史数据可能无分析结果关联
- 首次运行时 SqlSugar CodeFirst 会自动增加新字段，不影响已有数据
