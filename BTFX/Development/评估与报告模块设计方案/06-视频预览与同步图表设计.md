# 06 - 视频预览与同步图表设计

## 一、设计目标

本章节聚焦 **分析完成后的结果预览体验**，目标是在 Step4 分析阶段结束后，向用户提供一个可交互的“视频 + 曲线”联合预览界面。

核心目标如下：

1. **播放算法输出的标注视频**
   - 视频中已叠加关键点、骨骼连线、关节角数值等信息
   - 支持播放、暂停、拖拽、快进快退、变速播放

2. **同步展示关节角度曲线图**
   - 默认展示髋、膝、踝、骨盆等角度-时间曲线
   - 曲线图与视频进度严格同步
   - 曲线上可显示当前播放位置指示线

3. **支持双向联动定位**
   - 拖动视频进度条，曲线图同步跳转
   - 点击曲线图，视频同步跳转到对应时间点
   - 保证用户可以快速定位异常姿态、关键帧和步态事件

4. **为报告模块复用图表能力**
   - 预览阶段使用的 CSV 解析能力、图表构建能力、关键事件标记能力，应尽量复用到报告模块

---

## 二、设计范围

本设计覆盖以下内容：

- 标注视频加载与播放控制
- 关节角度 CSV 数据读取与缓存
- OxyPlot 曲线图构建与同步指示线更新
- Step4AnalyzeView 中视频预览子界面的布局
- 视频与图表的时间同步机制
- 步态事件标记线与关键帧定位
- 预览性能优化与异常处理

本设计**不包含**：

- Python 算法内部的视频绘制逻辑
- 原始视频采集逻辑
- 报告导出模板本身（见 04 文档）

## 三、输入与输出

### 3.1 输入来源

视频预览模块依赖分析结果表与输出目录中的文件：

```
AnalysisResult
├── AnnotatedVideoPath           ── 标注视频路径
├── AnalysisCsvFiles             ── 关节角度/关键点/速度等 CSV 文件列表
├── SummaryFilePath              ── summary.json（步态事件、质量信息）
└── QualityControl               ── 质量控制信息
```

其中当前功能最低必需输入为：

| 输入项 | 必填 | 用途 |
|--------|------|------|
| annotated_video.mp4 | ✓ | 播放算法标注视频 |
| joint_angles.csv | ✓ | 绘制关节角度-时间曲线 |
| summary.json | ✓ | 读取步态事件、质量信息 |
| keypoint_trajectory.csv | | 后续扩展轨迹图 |
| joint_angular_velocity.csv | | 后续扩展角速度图 |

### 3.2 输出结果

视频预览模块本身不写回新的算法结果文件，但会产生以下运行时输出：

- 构建好的 `PlotModel`
- 当前播放时间 `VideoCurrentTime`
- 当前选中事件或关键帧
- 供报告模块复用的图表导出对象（后续扩展）

## 五、CSV 数据格式设计

### 5.1 joint_angles.csv 推荐格式

建议算法输出使用“宽表”格式，便于一次读取整帧数据：

```csv
frame_index,time_s,hip_flexion_deg,knee_flexion_deg,ankle_dorsi_deg,pelvis_obliquity_deg
0,0.000,12.4,5.2,-2.1,1.6
1,0.033,12.8,5.7,-1.9,1.8
2,0.067,13.1,6.5,-1.5,1.7
...
```

### 5.2 字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| frame_index | int | 帧序号，从 0 开始 |
| time_s | double | 时间戳，单位秒 |
| hip_flexion_deg | double | 髋关节角度 |
| knee_flexion_deg | double | 膝关节角度 |
| ankle_dorsi_deg | double | 踝关节角度 |
| pelvis_obliquity_deg | double | 骨盆角度 |

### 5.3 内存模型定义

```csharp
public sealed class JointAngleFrame
{
    public int FrameIndex { get; init; }
    public double TimeSeconds { get; init; }
    public double? HipAngleDeg { get; init; }
    public double? KneeAngleDeg { get; init; }
    public double? AnkleAngleDeg { get; init; }
    public double? PelvisAngleDeg { get; init; }
}
```

### 5.4 解析规则

1. `time_s` 为主时间轴，不依赖 `frame_index` 反推时间
2. 缺失值允许为空，UI 中使用断线而不是补零
3. 按 `time_s` 升序排序
4. 若时间戳重复，保留第一条并记录日志警告
5. 若首帧时间不为 0，允许自动归一化为从 0 开始

### 5.5 数据校验规则

| 校验项 | 规则 | 失败处理 |
|--------|------|---------|
| 文件存在 | 必须存在 | 预览失败 |
| 列名完整 | 至少存在 frame_index + time_s + 1 条角度曲线 | 降级显示 |
| 时间递增 | 应基本递增 | 记录警告，尝试排序 |
| 数据量 | 至少 10 帧 | 提示结果不足 |
| 时长一致 | CSV 末尾时间与视频时长差异 < 0.2s | 提示轻微偏差，允许播放 |

## 六、步态事件与时间轴标记

### 6.1 事件来源

从 `summary.json` 中读取步态事件，例如：

```json
{
  "gait_events": [
    { "name": "initial_contact", "time_s": 0.42 },
    { "name": "toe_off", "time_s": 0.78 },
    { "name": "mid_stance", "time_s": 0.56 }
  ]
}
```

### 6.2 事件标记用途

步态事件在视频预览阶段可用于：

- 在曲线图上显示垂直虚线
- 在进度条上显示刻度点
- 在侧边信息区显示“当前最近事件”
- 供后续关键帧截图与报告引用

### 6.3 事件标记视觉规范

| 事件类型 | 颜色 | 线型 | 说明 |
|---------|------|------|------|
| initial_contact | 红色 | 虚线 | 初始接触 |
| toe_off | 橙色 | 虚线 | 足离地 |
| mid_stance | 蓝色 | 点划线 | 中间站立 |
| 当前时间 | 亮红色 | 实线 | 实时 Tracker |

## 八、会话初始化流程

### 8.1 初始化时机

分析成功并读取 `AnalysisResult` 后，自动执行：

```
InitializePreviewAsync(analysisResult)
│
├── 校验 annotated_video.mp4 是否存在
├── 查找 joint_angles.csv 是否存在
├── 读取 summary.json 中的 gait_events
├── 调用 JointAngleDataService.LoadFramesAsync
├── 构建 4 个 PlotModel
├── 初始化 VideoCurrentTime = 0
├── 初始化 PlaybackSpeed = 1.0
└── 切换界面到视频预览视图
```

### 8.2 初始化失败时的降级策略

| 失败项 | 降级方式 |
|--------|---------|
| 视频缺失 | 仅显示参数概览 + 错误提示 |
| CSV 缺失 | 仅显示视频，不显示曲线图 |
| 事件缺失 | 不显示事件标记，但允许预览 |
| 全部缺失 | 保留分析完成状态，提示无法进入预览 |

## 十、时间同步核心机制

### 10.1 单一时间源原则

整个界面应只有一个核心时间源：

```csharp
VideoCurrentTime : TimeSpan
```

任何组件发生变化，最终都应归一到对 `VideoCurrentTime` 的更新。

### 10.2 时间变更来源

```
TimeSyncSource
├── MediaTimer      ── 视频播放轮询更新
├── SliderDrag      ── 进度条拖拽
├── PlotClick       ── 曲线图点击
├── StepCommand     ── 快进/快退
└── Initialization  ── 初始化设置
```

### 10.3 同步流程

```
VideoCurrentTime 变更
│
├── 更新 CurrentTimeDisplay
├── 更新 Slider.Value
├── 更新 MediaElement.Position（必要时）
├── 更新所有 PlotModel 中的 Tracker 线
└── 刷新“当前最近事件”显示
```

### 10.4 防止循环更新

为避免以下循环：

- MediaElement.Position → ViewModel
- ViewModel → Slider
- Slider → ViewModel
- ViewModel → MediaElement

建议在时间更新时显式传入来源：

```csharp
private void UpdateVideoTime(TimeSpan time, TimeSyncSource source)
{
    if (_isInternalSync && source != TimeSyncSource.MediaTimer)
    {
        return;
    }

    _isInternalSync = true;
    try
    {
        VideoCurrentTime = NormalizeTime(time);
        TrackerLinePosition = VideoCurrentTime.TotalSeconds;
        CurrentTimeDisplay = BuildCurrentTimeDisplay(VideoCurrentTime, VideoDuration);

        if (source is not TimeSyncSource.MediaTimer)
        {
            _mediaElement.Position = VideoCurrentTime;
        }

        UpdateAllTrackerLines(TrackerLinePosition);
    }
    finally
    {
        _isInternalSync = false;
    }
}
```

### 10.5 定时轮询策略

使用 `DispatcherTimer` 每 33ms 轮询 `MediaElement.Position`：

- 约等于 30fps，同步体验较平滑
- 相比每帧监听，WPF 实现更简单稳定
- 用户拖拽期间暂停定时器，释放后恢复

```csharp
_mediaTimer.Interval = TimeSpan.FromMilliseconds(33);
_mediaTimer.Tick += (_, _) =>
{
    if (!IsPlaying || IsSeeking)
    {
        return;
    }

    UpdateVideoTime(_mediaElement.Position, TimeSyncSource.MediaTimer);
};
```

## 十一、曲线图构建设计

### 11.1 PlotModel 统一样式

每个关节图表采用一致的图表结构：

- 一个 `LineSeries` 显示关节角度曲线
- 一个 `LineAnnotation` 显示当前时间 Tracker 线
- 多个 `LineAnnotation` 显示步态事件线
- Y 轴显示角度(°)
- X 轴显示时间(s)

### 11.2 构建示例

```csharp
private PlotModel BuildJointPlot(
    string title,
    OxyColor lineColor,
    Func<JointAngleFrame, double?> selector,
    IReadOnlyList<JointAngleFrame> frames,
    IReadOnlyList<GaitEventMarker> events,
    double durationSeconds)
{
    var model = new PlotModel
    {
        Title = title,
        PlotMargins = new OxyThickness(48, 8, 16, 28),
        Padding = new OxyThickness(0),
        IsLegendVisible = false
    };

    model.Axes.Add(new LinearAxis
    {
        Position = AxisPosition.Bottom,
        Minimum = 0,
        Maximum = durationSeconds,
        Title = "t(s)",
        IsZoomEnabled = false,
        IsPanEnabled = false
    });

    model.Axes.Add(new LinearAxis
    {
        Position = AxisPosition.Left,
        Title = "°",
        IsZoomEnabled = false,
        IsPanEnabled = false
    });

    var series = new LineSeries
    {
        Color = lineColor,
        StrokeThickness = 1.8,
        LineStyle = LineStyle.Solid
    };

    foreach (var frame in frames)
    {
        var value = selector(frame);
        if (value.HasValue)
        {
            series.Points.Add(new DataPoint(frame.TimeSeconds, value.Value));
        }
        else
        {
            series.Points.Add(DataPoint.Undefined);
        }
    }

    model.Series.Add(series);
    model.Annotations.Add(CreateTrackerAnnotation());

    foreach (var evt in events)
    {
        model.Annotations.Add(CreateEventAnnotation(evt));
    }

    return model;
}
```

### 11.3 颜色建议

| 曲线 | 颜色 |
|------|------|
| 髋关节 | `#1976D2` |
| 膝关节 | `#2E7D32` |
| 踝关节 | `#EF6C00` |
| 骨盆角度 | `#7B1FA2` |
| 当前时间线 | `#E53935` |

### 11.4 点击定位

用户点击 PlotView 任意位置时，需要将屏幕 X 坐标转换为时间值：

```csharp
var xAxis = plotModel.Axes.First(a => a.Position == AxisPosition.Bottom);
var time = xAxis.InverseTransform(mouseX);
SeekToCommand.Execute(Math.Clamp(time, 0, VideoDurationSeconds));
```

应限制只接受绘图区点击，避免点到坐标轴标签区产生误判。

## 十二、MediaElement 与 View 的协作设计

### 12.1 为什么需要 View 层协作

WPF `MediaElement` 不是纯 MVVM 友好的控件，以下能力通常需要在 `Step4AnalyzeView.xaml.cs` 中处理：

- `MediaOpened` 获取视频总时长
- `MediaEnded` 处理播放结束
- `MediaFailed` 捕获加载失败
- 调用 `Play()`、`Pause()`、设置 `Position`

因此推荐采用“View 负责控件生命周期，ViewModel 负责状态”的折中方案。

### 12.2 View 侧职责

`Step4AnalyzeView.xaml.cs` 建议承担：

1. 将 `AnnotatedVideoPath` 设置到 `MediaElement.Source`
2. 监听媒体事件并回传 ViewModel
3. 在 ViewModel 命令触发时执行 `Play()` / `Pause()`
4. 处理 `Slider` 拖拽开始/结束事件
5. 处理 PlotView 鼠标点击转时间

### 12.3 View 与 ViewModel 交互方式

可选两种方式：

#### 方案 A：代码后置直接桥接（推荐）

- 简单直接
- 与 WPF `MediaElement` 的使用习惯一致
- 更适合当前项目规模

#### 方案 B：Behavior/AttachedProperty 封装

- 更纯粹
- 初期实现成本较高
- 适合后续统一沉淀为控件库

当前建议先采用 **方案 A**。

## 十三、关键交互场景设计

### 13.1 正常播放

```
用户点击播放
│
├── View 调用 MediaElement.Play()
├── ViewModel.IsPlaying = true
├── DispatcherTimer 开始轮询 Position
├── Slider 实时移动
└── 4 个 PlotModel 的 Tracker 线同步移动
```

### 13.2 拖拽进度条

```
用户开始拖拽 Slider
│
├── IsSeeking = true
├── 暂停 Timer 轮询
├── Slider.Value 变化 → Seek 到目标时间
├── Tracker 线实时更新
└── 拖拽结束后恢复 Timer
```

建议拖拽时视频不必逐像素刷新，可在拖拽完成时正式 Seek，一次性跳转，减少卡顿。

### 13.3 点击曲线图定位

```
用户点击某曲线图 X=1.84s 位置
│
├── 将点击位置映射为时间值
├── VideoCurrentTime = 1.84s
├── MediaElement.Position = 1.84s
├── Slider.Value = 1.84
└── Tracker 线移动到 1.84s
```

建议点击曲线图时：

- 若当前正在播放，则先暂停
- 定位完成后等待用户继续播放

这样更符合临床查看异常姿态时的使用习惯。

### 13.4 快进快退

- 默认步长：`1.0s`
- 视频总时长小于 3 秒时，自动降级为 `0.25s`
- 到达边界时自动截断到 `[0, duration]`

### 13.5 播放到结尾

- 自动暂停
- `VideoCurrentTime = VideoDuration`
- Slider 停在末端
- Tracker 线停在最后一帧
- 播放按钮切回“重新播放”语义

## 十四、性能设计

### 14.1 基本原则

视频播放与图表同步必须避免以下问题：

- 每 33ms 重建整个 PlotModel
- 每次时间变化都重新读取 CSV
- 在 UI 线程中执行大规模数据解析
- 拖拽过程中频繁触发高成本渲染

### 14.2 推荐策略

1. **CSV 只加载一次**
   - 初始化时读取并缓存为 `JointAngleFrames`

2. **PlotModel 只构建一次**
   - 仅更新 Tracker 线的 `X` 值
   - 使用 `InvalidatePlot(false)` 局部刷新

3. **统一使用秒作为内部时间单位**
   - 避免 TimeSpan/double 来回转换产生复杂度

4. **限制默认展示曲线数量**
   - 初版只展示 4 条曲线
   - 后续扩展速度/角速度时，采用分页或折叠面板

5. **拖拽时节流**
   - Slider 连续变更时可按 50~100ms 节流更新 Tracker

### 14.3 大文件场景优化

当视频超过 2 分钟、CSV 超过 3600 帧时，可采用：

- 图表降采样显示
- 仅保留关键显示精度
- 报告导出再使用全量数据

当前步态分析视频通常较短，初版可先不引入复杂降采样算法，但需在接口设计中预留扩展点。

## 十五、异常处理设计

### 15.1 预览阶段异常分类

| 场景 | 说明 | UI 处理 |
|------|------|---------|
| 视频文件不存在 | AnnotatedVideoPath 无效 | 提示“标注视频缺失” |
| 视频加载失败 | 编码不兼容/文件损坏 | 提示“视频无法播放” |
| CSV 文件缺失 | joint_angles.csv 不存在 | 降级为仅视频预览 |
| CSV 列名错误 | 无法解析时间或角度字段 | 提示“曲线数据格式异常” |
| 视频与 CSV 时长不一致 | 时间轴轻微偏移 | 显示黄色警告 |
| PlotView 点击转换失败 | 坐标不在绘图区 | 忽略点击 |

### 15.2 错误提示建议

```
预览资源加载失败
├── 标注视频缺失：请重新分析或检查输出目录
├── 曲线数据缺失：仅展示视频，无法显示同步曲线
├── 视频格式不支持：请确认算法输出为标准 MP4(H.264)
└── 数据格式异常：请联系技术支持并附上输出目录日志
```

### 15.3 日志记录建议

建议记录以下关键日志：

- 预览初始化开始/结束
- 视频路径、CSV 路径、事件数量
- CSV 行数、首尾时间、是否存在缺失值
- 视频总时长与 CSV 总时长差值
- 用户定位操作（仅 Debug 级别）
- 视频加载失败异常详情

## 二十、结论

视频预览与同步图表功能是 Step4 分析阶段的核心体验增强点。其本质不是简单的视频播放，而是围绕“**时间轴统一**”构建的结果解释界面。

本设计建议以 `VideoCurrentTime` 作为唯一时间事实源，通过：

- `MediaElement` 负责视频呈现
- `Slider` 负责直观定位
- `OxyPlot` 负责曲线解释
- `IVideoPreviewService` / `IJointAngleDataService` 负责解耦服务能力

最终形成一个可播放、可定位、可解释、可复用到报告模块的完整预览方案。

后续如进入实现阶段，可基于本设计继续细化：

- `Step4AnalyzeView.xaml` 具体控件树
- `Step4AnalyzeView.xaml.cs` 的媒体桥接代码
- `Step4AnalyzeViewModel` 的命令与状态机实现
- `JointAngleDataService` 的 CSV 解析与 PlotModel 工厂实现
