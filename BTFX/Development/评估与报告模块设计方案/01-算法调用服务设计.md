# 01 - 算法调用服务设计

## 一、服务职责

`IGaitAnalysisService` / `GaitAnalysisService` 是评估模块的核心服务，负责：

1. **构建算法输入**：将测量记录、患者信息、设备配置组装为符合通讯协议的 JSON 配置文件
2. **启动算法进程**：通过 `Process` 启动 `gait_analysis.exe`，传入配置文件路径
3. **实时状态监听**：逐行读取 stdout 输出的 JSON 状态消息，解析并通过事件/回调通知 UI
4. **结果文件解析**：读取 `summary.json` 和 CSV 文件，转化为应用内模型
5. **进程生命周期管理**：超时控制、取消支持、异常退出处理

---

## 二、接口设计

```
IGaitAnalysisService
├── Task<AnalysisResult> RunAnalysisAsync(AnalysisRequest request, CancellationToken ct)
│     ── 主方法：执行完整分析流程，返回结果
│
├── event EventHandler<AnalysisProgressEventArgs> ProgressChanged
│     ── 进度变更事件（progress %, 阶段描述, 状态）
│
├── event EventHandler<AnalysisLogEventArgs> LogReceived
│     ── 日志消息事件（用于任务抽屉显示）
│
├── Task CancelCurrentAnalysisAsync()
│     ── 取消当前正在运行的分析任务
│
├── bool IsAnalysisRunning { get; }
│     ── 当前是否有分析任务在运行
│
└── Task<bool> ValidateEnvironmentAsync()
      ── 验证算法环境（exe 是否存在、版本是否匹配等）
```

### 2.1 AnalysisRequest（请求参数）

```
AnalysisRequest
├── MeasurementRecord Record       ── 测量记录（含视频路径）
├── Patient Patient                ── 患者信息（身高、体重等）
├── DeviceConfig DeviceConfig      ── 设备配置（相机距离等）
├── AnalysisOptions Options        ── 分析选项（哪些参数需要计算）
└── string OutputDirectory         ── 输出目录路径
```

### 2.2 AnalysisProgressEventArgs（进度事件参数）

```
AnalysisProgressEventArgs
├── string RequestId               ── 请求编号
├── string TaskStatus              ── 状态（pending/running/completed/failed）
├── int Progress                   ── 进度百分比（0-100）
├── string Message                 ── 状态描述文字
├── int? ErrorCode                 ── 错误码（失败时）
└── DateTime Timestamp             ── 时间戳
```

## 三、核心流程

### 3.1 RunAnalysisAsync 主流程

```
RunAnalysisAsync(request, cancellationToken)
│
├── [1] 验证输入
│     ├── 检查视频文件是否存在
│     ├── 检查患者身高是否已填（必填）
│     └── 检查相机距离是否已配置（必填）
│
├── [2] 构建配置文件
│     ├── 生成 request_id（格式：GAIT_{yyyyMMdd}_{序号}）
│     ├── 组装 JSON 配置（严格遵循通讯协议格式）
│     ├── 确定输出目录（按文件目录规范）
│     └── 写入 task_config.json 到输出目录
│
├── [3] 启动算法进程
│     ├── 配置 ProcessStartInfo
│     │    ├── FileName = 算法 exe 路径
│     │    ├── Arguments = --config "配置文件路径"
│     │    ├── RedirectStandardOutput = true
│     │    ├── RedirectStandardError = true
│     │    └── CreateNoWindow = true
│     ├── 注册 OutputDataReceived 回调
│     ├── 注册 ErrorDataReceived 回调
│     └── 启动进程 + BeginOutputReadLine
│
├── [4] 等待完成
│     ├── WaitForExitAsync + CancellationToken
│     ├── 超时处理（可配置，默认 10 分钟）
│     └── 取消处理（Kill 进程）
│
├── [5] 判断结果
│     ├── 检查退出码
│     │    ├── 0 = 成功
│     │    ├── 1 = 配置文件错误
│     │    ├── 2 = 输入文件不存在
│     │    ├── 3 = 视频读取失败
│     │    ├── 4 = 分析处理失败
│     │    ├── 5 = 结果导出失败
│     │    └── 9 = 未知错误
│     └── 读取 summary.json 验证 success 字段
│
├── [6] 解析结果（成功时）
│     ├── 反序列化 summary.json → AnalysisSummary
│     ├── 提取步态事件参数 → GaitEventParameters
│     ├── 提取运动学汇总 → KinematicSummary
│     ├── 记录 CSV 文件路径 → CsvFileInfo
│     ├── 记录标注视频路径 → AnnotatedVideoPath（算法输出的叠加关键点/连线/关节角的视频）
│     ├── 验证标注视频文件存在且可播放
│     └── 提取质量控制信息 → QualityControl
│
└── [7] 返回 AnalysisResult（含 AnnotatedVideoPath）
```

### 3.2 stdout 状态解析流程

```
OutputDataReceived 回调
│
├── 过滤空行
├── 尝试 JSON 反序列化为 TaskStatusMessage
│    ├── 成功
│    │    ├── 检查 type == "status"
│    │    ├── 映射 task_status → UI 状态
│    │    │    ├── "pending"    → 显示"任务已接受"
│    │    │    ├── "running"    → 更新进度条 + 阶段文字
│    │    │    ├── "completed"  → 显示"分析完成"
│    │    │    ├── "failed"     → 显示错误信息 + 错误码
│    │    │    └── "cancelled"  → 显示"任务已取消"
│    │    ├── 触发 ProgressChanged 事件
│    │    └── 触发 LogReceived 事件（记录到任务抽屉）
│    │
│    └── 失败（非 JSON）
│         └── 作为普通日志行处理，触发 LogReceived
```

## 四、配置文件生成细节

### 4.1 字段映射关系

| 配置文件字段 | 数据来源 | 说明 |
|-------------|---------|------|
| `request_id` | 自动生成 | 格式 `GAIT_{yyyyMMdd}_{3位序号}` |
| `protocol_version` | Constants | 固定 "V2.0" |
| `algorithm_version` | AppSettings | 算法版本号 |
| `model_version` | AppSettings | 模型版本号 |
| `task_type` | 固定 | "gait_analysis" |
| `analysis_mode` | 固定 | "offline" |
| `subject_info.subject_id` | Patient.Id | 格式化为 "P{Id:D4}" |
| `subject_info.gender` | Patient.Gender | "male" / "female" |
| `subject_info.age` | Patient.BirthDate | 计算年龄 |
| `subject_info.height_m` | Patient.Height | **注意：Patient 存 cm，需转换为 m** |
| `subject_info.weight_kg` | Patient.Weight | 直接取值 |
| `video_info.sagittal_video_path` | MeasurementRecord.SideVideoPath | 侧面=矢状面 |
| `video_info.coronal_video_path` | MeasurementRecord.FrontVideoPath | 正面=冠状面 |
| `video_info.video_fps` | VideoSpec 推算 | 30 或其他 |
| `video_info.video_resolution` | VideoSpec 推算 | "1920x1080" 等 |
| `device_info.side_camera_to_walkway_distance_m` | DeviceConfig | **必填**，需设置页配置 |
| `analysis_options.*` | 用户选择 / 默认全选 | 4 个布尔开关 |

### 4.2 算法输出文件说明

算法除了输出 `summary.json` 和 CSV 时间序列数据外，还会输出一个**与原视频同样时长的标注视频**：

| 输出文件 | 说明 |
|---------|------|
| `summary.json` | 分析结果汇总（步态事件参数、运动学汇总、CSV路径、标注视频路径、质量控制） |
| `annotated_video.mp4` | **标注视频**：在原视频画面上叠加绘制了识别到的人体关键点、骨骼连线、关节角度数值 |
| `joint_angle_timeseries.csv` | 关节角度时间序列 |
| `keypoint_trajectory.csv` | 关键点运动轨迹 |
| `keypoint_velocity.csv` | 关键点速度 |
| `joint_angular_velocity.csv` | 关节角速度 |
| `debug.log` | 算法调试日志 |

`summary.json` 中新增字段 `annotated_video_path` 指向标注视频的文件路径。

### 4.3 关键校验

- **Patient.Height 必须有值**（算法必填）
- **相机距离必须已配置**（算法必填）
- **视频文件必须物理存在**

## 五、进程管理策略

### 5.1 超时机制
- 默认超时 **10 分钟**，可在设置中配置
- 使用 `CancellationTokenSource.CreateLinkedTokenSource` 结合外部取消和超时
- 超时后先尝试 `CloseMainWindow()`，等待 5 秒后 `Kill()`

### 5.2 取消机制
- 用户点击"取消分析"按钮
- 调用 `CancelCurrentAnalysisAsync()`
- 向进程发送取消信号（Kill）
- UI 显示"分析已取消"

### 5.3 重入保护
- 使用 `SemaphoreSlim(1,1)` 确保同一时间只有一个分析任务运行
- 如果已有任务运行，新请求应提示用户

### 5.4 进程异常退出
- 如果进程非正常退出（崩溃）
- 检查是否有 summary.json（可能部分完成）
- 读取 stderr 输出作为错误信息
- 读取 debug.log 获取额外诊断信息

---

## 六、线程安全考虑

- `OutputDataReceived` 回调在**非 UI 线程**触发
- 所有 UI 更新（进度条、状态文本）需通过 `Dispatcher.InvokeAsync` 回到 UI 线程
- 当前项目已有 `App.IsShuttingDown` 的应用关闭判断机制，需在回调中检查
- 事件触发使用 `SynchronizationContext` 或直接 `Dispatcher` 回调

## 七、设备配置扩展

需要在设置模块中新增算法相关配置项：

```
AlgorithmSettings（新增）
├── ExePath           ── 算法程序路径（默认 Algorithm/gait_analysis.exe）
├── TimeoutMinutes    ── 分析超时时间（默认 10）
├── AlgorithmVersion  ── 算法版本号
├── ModelVersion      ── 模型版本号
└── SideCameraDistance ── 侧向相机与步道距离（m）（**关键参数**）
```

这些配置需要在 **设置 → 设备配置** 页面中提供编辑入口。

## 八、与 MeasurementViewModel 的衔接

```
MeasurementViewModel (Step4)
│
├── 用户点击"开始分析"
│     └── StartAnalyzeCommand
│           ├── 校验前置条件（双视频、患者信息完整性）
│           ├── 构建 AnalysisRequest
│           ├── 调用 IGaitAnalysisService.RunAnalysisAsync()
│           ├── 订阅 ProgressChanged → 更新 AnalysisProgress, 任务日志
│           └── 等待结果
│
├── 分析完成
│     ├── 保存结果到数据库（含标注视频路径）
│     ├── 更新 MeasurementRecord.Status = Completed
│     ├── 更新分析阶段标记
│     └── 自动进入视频预览状态（加载标注视频 + 同步曲线图）
│
└── 用户点击"取消分析"
      └── CancelAnalyzeCommand
            └── IGaitAnalysisService.CancelCurrentAnalysisAsync()
```
