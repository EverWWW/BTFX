# 03 - 评估模块 UI 与交互设计

## 一、定位

评估模块对应测量向导的 **第4步（Step4：分析评估）**。用户在完成视频导入（Step2）和参数确认（Step3）后进入此步骤，触发算法分析并查看结果。

---

#### 2.4.2 核心交互设计

**A. 视频播放区域（左侧）**
- 使用 WPF `MediaElement` 播放算法输出的标注视频（`annotated_video.mp4`）
- 视频中已包含算法绘制的关键点标记、骨骼连线、关节角度数值
- 视频尺寸自适应，保持原始宽高比，背景填充深灰色

**B. 进度条控制区（右侧上方）**
- **进度条（Slider）**：与视频播放进度实时同步
  - 拖拽进度条 → 视频跳转到对应位置（Seek）
  - 视频播放时 → 进度条实时跟随移动
  - 进度条下方标注时间刻度（0s, 1s, 2s, ...）
- **时间显示**：`当前时间 / 总时长` 格式，如 `00:01.23 / 00:05.60`
- **播放/暂停按钮**：切换播放状态
- **快退/快进按钮**：跳转 ±1 秒（或 ±0.5 秒，可配置）
- **播放速度控制**：
  - 提供按钮组：`0.25x` `0.5x` `1x` `1.5x` `2x`
  - 当前选中的速度高亮显示
  - 默认 `1x` 正常速度
  - 通过 `MediaElement.SpeedRatio` 实现

**C. 同步曲线图区域（右侧下方，核心联动机制）**
- 多个 OxyPlot `PlotView` **纵向堆叠**，每个图表显示一个关节的角度-时间曲线
- 默认显示 4 个关节：髋关节、膝关节、踝关节、骨盆
- **关键同步机制**：
  - 所有曲线图的**横轴（时间轴）范围**与**进度条宽度**完全对齐
  - 曲线图上有一条**垂直指示线（Tracker Line）**，位置与当前视频播放时间同步
  - 拖拽进度条 → 指示线跟随移动 → 视频跳转
  - 视频播放中 → 指示线实时跟随 → 进度条同步
  - 点击曲线图某处 → 视频跳转到该时间点 → 进度条同步
- 曲线图**不可缩放/平移**（横轴固定对应视频全长），确保与进度条始终对齐

**D. 进度条与曲线图的对齐实现要点**
```
布局约束（XAML）：
├── 进度条 Slider 的宽度 = 右侧面板宽度 - 左右边距
├── 每个 PlotView 的宽度 = 与 Slider 完全等宽（Binding 或共享 Grid 列）
├── PlotView 的 PlotAreaBounds.Left 对齐 Slider 的 Track 起始位置
├── PlotView 的 PlotAreaBounds.Right 对齐 Slider 的 Track 结束位置
└── 实现方式：将 Slider 和所有 PlotView 放在同一个 Grid 列中
    使用 OxyPlot 的 Padding/PlotMargins 精确控制绘图区左右边距与 Slider Track 对齐
```

#### 2.4.3 时间同步机制详细设计

```
时间同步核心逻辑：

1. 单一时间源：VideoCurrentTime (TimeSpan) —— ViewModel 中的核心属性
   ├── 由 MediaElement.Position 驱动更新（DispatcherTimer 每 33ms 采样一次，约 30fps）
   ├── 由 Slider 拖拽事件更新（用户交互时暂停 Timer 采样）
   └── 由曲线图点击事件更新

2. 联动绑定关系：
   VideoCurrentTime 变更 →
   ├── Slider.Value 更新（TwoWay Binding，单位秒）
   ├── MediaElement.Position 更新（仅当来源不是 MediaElement 时）
   ├── 所有曲线图的 Tracker 垂直线位置更新（通过 OxyPlot Annotation）
   └── 当前时间文本更新（格式化为 mm:ss.ff）

3. 防抖策略：
   ├── Slider 拖拽中：暂停 Timer → 仅更新视频位置和 Tracker → 释放后恢复 Timer
   ├── 曲线图点击：暂停视频 → 跳转时间 → 用户手动恢复播放
   └── 播放速度变更：更新 SpeedRatio，Timer 采样频率不变
```

#### 2.4.4 交互要点总结

| 操作 | 响应 |
|------|------|
| 点击"播放" | 视频开始播放，进度条和 Tracker 线实时跟随 |
| 点击"暂停" | 视频暂停，进度条和 Tracker 线停在当前位置 |
| 拖拽进度条 | 视频 Seek 到对应时间，Tracker 线同步移动 |
| 点击曲线图 | 视频暂停并 Seek 到该时间，进度条同步 |
| 切换播放速度 | 视频变速播放，其余联动不变 |
| 点击"快退/快进" | 视频向前/后跳转 1 秒，进度条和 Tracker 同步 |
| 视频播放到末尾 | 自动暂停，停在最后一帧 |
| 点击"参数概览" | 切换到参数卡片视图（原状态C设计），保留视频预览入口 |
| 点击"查看报告" | 跳转到报告模块 |
| 点击"重新分析" | 确认后重新分析，清除当前预览 |

## 四、与 MeasurementViewModel 的关系

Step4AnalyzeViewModel **不是独立的页面 ViewModel**，而是作为 `MeasurementViewModel` 的**子模块/逻辑部分**存在（与当前项目架构一致——Step4 内嵌在测量向导中）。

有两种实现方式可选：

### 方案A：内嵌在 MeasurementViewModel（当前架构风格）

将分析相关的属性和命令直接放在 `MeasurementViewModel` 中，与 Step1/Step2 字段并列。

**优点**：简单，与现有架构一致
**缺点**：MeasurementViewModel 会更加膨胀

### 方案B：独立 ViewModel + 组合（推荐）

创建独立的 `Step4AnalyzeViewModel`，由 `MeasurementViewModel` 持有并传递上下文。

```csharp
// MeasurementViewModel 中
public Step4AnalyzeViewModel AnalyzeViewModel { get; }

// Step4AnalyzeView.xaml 中
DataContext="{Binding AnalyzeViewModel}"
```

**优点**：职责分离，便于维护和测试
**缺点**：需要双向通信（通过事件或回调）

**建议采用方案B**，原因：
1. 分析逻辑相对独立且复杂
2. 便于后续扩展（例如支持批量分析）
3. 独立 ViewModel 更容易做单元测试

## 七、动画与过渡效果建议

1. **状态切换**：使用 Fade 过渡动画（300ms），避免突兀跳变
2. **进度条**：使用平滑动画更新（不跳变），Material Design 风格
3. **参数卡片**：分析完成时使用 Stagger 动画逐个显示
4. **质量指标**：使用颜色渐变指示（绿色=好，黄色=一般，红色=差）
5. **视频预览进入**：分析完成后平滑过渡到视频预览界面（Slide + Fade，500ms）
6. **Tracker 线**：实时跟随视频播放位置，平滑移动（不使用动画，直接绑定时间值避免延迟）
7. **视频/参数视图切换**：使用左右 Slide 动画（300ms），给用户"翻页"的感觉

## 八、视频预览技术选型与注意事项

### 8.1 MediaElement 使用要点
- 使用 WPF 内置 `MediaElement`，`LoadedBehavior="Manual"` 模式
- 通过 `DispatcherTimer`（33ms 间隔，约 30fps）轮询 `MediaElement.Position` 更新 ViewModel
- Seek 操作：设置 `MediaElement.Position = new TimeSpan(...)`
- 变速播放：设置 `MediaElement.SpeedRatio`（WPF 内置支持）
- 视频加载：`MediaElement.Source = new Uri(annotatedVideoPath)`

### 8.2 OxyPlot 同步曲线图要点
- 每个关节一个独立的 `PlotView`，共享相同的 X 轴范围（0 到视频总时长秒数）
- 使用 `LineAnnotation { Type = LineAnnotationType.Vertical }` 作为 Tracker 指示线
- Tracker 线颜色建议使用醒目的红色（#FF0000），线宽 1.5px
- 曲线颜色使用主题色系区分不同关节（髋=蓝色、膝=绿色、踝=橙色、骨盆=紫色）
- 禁用 PlotView 的 Pan/Zoom 手势（`Controller = new PlotController()` 清空手势）
- 点击事件：`PlotView.MouseDown` → 将鼠标 X 坐标映射回时间值 → 更新 VideoCurrentTime

### 8.3 进度条与曲线图精确对齐
- 将 Slider 和 PlotView 放在**同一 Grid 列**中
- Slider 使用自定义 Template，确保 Track 部分没有额外内边距
- PlotView 通过 `PlotModel.Padding = new OxyThickness(leftPad, 0, rightPad, 0)` 精确调整绘图区左右边界
- 或使用 `PlotModel.PlotMargins` 设置固定边距，使 Y 轴标签不影响绘图区宽度
- 建议 Y 轴标签宽度固定为 40px，则 PlotView 左边距 = 40px，Slider 左边距也 = 40px
