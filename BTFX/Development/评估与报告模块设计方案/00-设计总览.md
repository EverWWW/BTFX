## 一、背景与目标

### 1.1 背景

步态分析算法已由 Python 团队完成并封装为本地可执行程序（`gait_analysis.exe`），通过命令行方式接收 JSON 配置文件、输出 JSON 结果文件和 CSV 时间序列数据。

当前 BTFX 上位机中：
- **测量评估模块**（Step4 分析阶段）是一个空壳 `Step4AnalyzeViewModel`，无实际逻辑
- **报告模块**已有基础框架（列表管理、PDF 导出、打印），但缺乏与算法结果的对接

### 1.2 目标

1. **评估模块**：在测量向导的第4步（分析评估），调用 Python 算法进程，实时显示进度，解析分析结果并存储
2. **视频预览**：分析完成后自动进入视频预览阶段，播放算法输出的标注视频（含关键点、连线、关节角），同步展示多条关节角度曲线图，支持进度拖拽和播放速度控制
3. **报告模块**：读取分析结果数据，生成包含步态事件参数、运动学参数和曲线图表的完整评估报告

### 1.3 核心约束

- 算法调用方式：**本地进程调用**（Process 启动 exe）
- 通讯方式：**单行 JSON stdout** 传递实时状态
- 结果交互：算法写出 `summary.json` + CSV 文件，上位机读取
- 所有单位：时间(s)、长度(m)、角度(°)、速度(m/s)、角速度(°/s)

## 四、数据流全景

```
用户点击"开始分析"
       │
       ▼
[1] 上位机构建 task_config.json
    ├── 从 MeasurementRecord 取视频路径
    ├── 从 Patient 取受试者信息（身高、体重等）
    ├── 从 DeviceConfig 取设备信息（相机距离等）
    └── 生成 request_id、设置输出目录
       │
       ▼
[2] 启动 gait_analysis.exe --config "task_config.json"
       │
       ▼
[3] 实时读取 stdout 状态消息（单行 JSON）
    ├── pending   → 更新状态："任务已接受"
    ├── running   → 更新进度条 + 阶段文字
    ├── completed → 读取结果文件
    └── failed    → 显示错误信息
       │
       ▼
[4] 读取 summary.json
    ├── 步态事件参数 → 写入 GaitParameters 表
    ├── 运动学汇总   → 写入 KinematicSummary 表（新增）
    ├── CSV 路径     → 记录到 AnalysisResult 表（新增）
    ├── 标注视频路径 → 记录到 AnalysisResult 表（新增）
    └── 质量控制信息 → 记录并提示
       │
       ▼
[5] 进入视频预览阶段（新增核心流程）
    ├── 加载算法输出的标注视频（含关键点、骨骼连线、关节角叠加绘制）
    ├── 加载关节角度 CSV → 生成同步曲线图（髋/膝/踝/骨盆）
    ├── 视频播放器 + 进度条 + 曲线图横轴同步联动
    ├── 用户可拖拽进度条定位、调节播放速度
    └── 预览完成后可继续查看参数汇总或生成报告
       │
       ▼
[6] 读取 CSV 文件（按需）
    ├── 关节角度时间序列 → 绘制角度-时间曲线（视频预览同步 + 报告图表）
    ├── 关键点轨迹      → 绘制运动轨迹图
    ├── 关键点速度      → 绘制速度曲线
    └── 关节角速度      → 绘制角速度曲线
       │
       ▼
[7] 更新 MeasurementRecord 状态为 Completed
       │
       ▼
[8] 用户可进入报告模块，选择此次测量 → 自动填充分析数据 → 生成报告
```

## 五、关键设计决策

### 5.1 算法程序路径管理
- 算法 exe 路径通过**设置页面配置**，存入 AppSettings
- 默认路径为应用程序目录下 `Algorithm/gait_analysis.exe`
- 启动前校验文件是否存在

### 5.2 文件目录规范
```
应用数据根目录/
├── Data/
│   ├── Database/
│   │   └── BTFX.db
│   ├── Measurements/          ← 测量数据根目录
│   │   └── {PatientId}/
│   │       └── {MeasurementId}/
│   │           ├── front.mp4
│   │           ├── side.mp4
│   │           └── Analysis/  ← 分析结果目录
│   │               └── {RequestId}/
│   │                   ├── task_config.json
│   │                   ├── summary.json
│   │                   ├── annotated_video.mp4      ← 标注视频（关键点+连线+关节角叠加）
│   │                   ├── joint_angle_timeseries.csv
│   │                   ├── keypoint_trajectory.csv
│   │                   ├── keypoint_velocity.csv
│   │                   ├── joint_angular_velocity.csv
│   │                   └── debug.log
│   └── Reports/
│       └── {ReportNumber}.pdf
└── Algorithm/
    └── gait_analysis.exe
```

### 5.3 单位转换策略
- 算法输出单位固定（m, s, °）
- 上位机显示时根据用户设置的单位偏好进行转换（例如 m→cm）
- 存储时**统一使用算法原始单位**，仅在展示层转换

### 5.4 一次测量 vs 多次分析
- 一条 MeasurementRecord 可以关联**多次分析结果**（重新分析场景）
- 报告生成时默认使用**最新一次成功的分析结果**
- 历史分析结果保留，用户可查看对比

## 七、开发优先级建议

```
Phase 1 ── 算法调用核心（可独立测试）
  ├── 数据模型定义（输入/输出 JSON 映射，含标注视频路径）
  ├── GaitAnalysisService 实现
  └── 单元测试（模拟 exe 调用）

Phase 2 ── 评估模块 UI（分析流程）
  ├── Step4AnalyzeView 界面开发
  ├── Step4AnalyzeViewModel 实现
  └── 进度展示 + 错误处理

Phase 3 ── 视频预览与同步曲线（核心新功能）
  ├── 标注视频播放器（MediaElement）
  ├── 关节角度曲线图区域（OxyPlot，多图堆叠）
  ├── 视频进度条 ↔ 曲线图横轴同步联动机制
  ├── 播放速度控制（0.25x/0.5x/1x/1.5x/2x）
  ├── 进度条拖拽定位
  └── VideoPreviewViewModel + VideoPreviewView

Phase 4 ── 数据入库
  ├── 分析结果写入数据库（含标注视频路径）
  ├── GaitParameters 扩展字段迁移
  └── 新表创建（AnalysisResult, KinematicSummary）

Phase 5 ── 报告模块增强
  ├── CSV 数据读取 + 图表生成
  ├── ReportPreviewHelper 增强
  ├── ReportPdfExporter 增强
  └── 报告模板更新

Phase 6 ── 打磨与优化
  ├── 异常处理完善（视频文件缺失/损坏）
  ├── 质量控制提示
  ├── 单位转换
  └── 多次分析对比
```
