- 影响用户体验

```xaml

<!-- 全局DataGridRow样式 - 去除焦点边框 -->
```

```csharp
/// 初始化数据加载
/// </summary>
private async Task InitializeDataAsync()
{
    try
    {
        // 如果有权限，自动加载用户和科室列表
        if (ShowUserManagementTab)
        {
            await LoadUsersAsync();

        if (ShowDepartmentTab)

    }
```

### 问题3：语言和主题切换未完全生效 ✅ 已修复

**问题描述**：
- 切换语言后，只有登录界面和标题栏文字生效
- 导航界面和其他已打开的界面没有立即更新
- 切换主题后视觉效果没有立即应用到所有界面

**根本原因**：
- 资源字典更新后，已创建的UI元素不会自动刷新绑定
- WPF需要强制触发UI更新才能应用新的资源

**修复方案**：

#### 1. LocalizationService强制刷新UI

在 `BTFX\Services\Implementations\LocalizationService.cs` 中添加强制刷新逻辑：

```csharp
/// <summary>
/// 应用指定语言
/// </summary>
public void ApplyLanguage(AppLanguage language)
{
    // ... 原有加载资源代码 ...
    
    try
    {
        var newDict = new ResourceDictionary { Source = resourceUri };
        Application.Current.Resources.MergedDictionaries.Add(newDict);

        CurrentLanguage = language;
        
        // 触发事件通知UI更新
        LanguageChanged?.Invoke(this, language);
        
        // 强制刷新所有打开的窗口
        Application.Current.Dispatcher.Invoke(() =>
        {
            foreach (Window window in Application.Current.Windows)
            {
                try
                {
                    // 触发资源更新
                    window.UpdateLayout();
                    
                    // 刷新DataContext以触发绑定更新
                    var dc = window.DataContext;
                    window.DataContext = null;
                    window.DataContext = dc;
                }
                catch
                {
                    // 忽略单个窗口刷新失败
                }
            }
        });
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"加载语言资源失败: {ex.Message}");
    }
}
```

#### 2. ThemeService强制刷新UI

在 `BTFX\Services\Implementations\ThemeService.cs` 中添加强制刷新逻辑：

```csharp
/// <summary>
/// 应用指定主题
/// </summary>
public void ApplyTheme(AppTheme theme)
{
    CurrentTheme = theme;

    var paletteHelper = new PaletteHelper();
    var currentTheme = paletteHelper.GetTheme();

    // 设置基础主题
    currentTheme.SetBaseTheme(theme == AppTheme.Dark ? BaseTheme.Dark : BaseTheme.Light);

    // 应用主题
    paletteHelper.SetTheme(currentTheme);

    // 触发事件
    ThemeChanged?.Invoke(this, theme);
    
    // 强制刷新所有打开的窗口
    Application.Current.Dispatcher.Invoke(() =>
    {
        foreach (Window window in Application.Current.Windows)
        {
            try
            {
                // 触发资源更新
                window.UpdateLayout();
                window.InvalidateVisual();
            }
            catch
            {
                // 忽略单个窗口刷新失败
            }
        }
    });
}
```

**刷新机制**：

1. **UpdateLayout()**：强制重新布局所有UI元素
2. **InvalidateVisual()**：标记视觉树需要重绘
3. **DataContext重置**：触发所有数据绑定重新计算（仅语言切换）

**影响范围**：

- 所有包含DynamicResource绑定的UI元素

## 技术细节

### DataGrid焦点边框移除原理

WPF的DataGrid默认在单元格获得焦点时会显示黑色边框（FocusVisualStyle）。解决方法：

1. 设置 `FocusVisualStyle="{x:Null}"` 移除焦点可视样式
2. 设置 `BorderThickness="0"` 移除边框
3. 在IsSelected触发器中同样设置BorderThickness="0"

### UI强制刷新机制

WPF的资源系统在ResourceDictionary更新后不会自动刷新已创建的UI元素，需要：

1. **UpdateLayout()**：
   - 强制执行布局过程
   - 重新计算所有元素的尺寸和位置
   - 触发Measure和Arrange

2. **InvalidateVisual()**：
   - 标记视觉树为"无效"
   - 触发重绘渲染
   - 应用新的样式和颜色

3. **DataContext重置**：
   - 设置为null再恢复
   - 触发所有Binding重新计算
   - 更新所有DynamicResource引用

### 异步加载最佳实践

在构造函数中使用 `_ = InitializeDataAsync()` 的优点：

1. **非阻塞**：构造函数立即返回，不会阻塞UI线程
2. **异常安全**：未等待的Task异常会被全局异常处理器捕获
3. **用户体验**：界面立即显示，数据在后台加载

## 已知限制

### 1. 语言切换限制

某些动态生成的UI元素可能需要重新创建才能应用新语言：
- 动态生成的对话框
- 运行时创建的控件
- 某些第三方控件

**解决方案**：这些元素通常在下次创建时会自动使用新语言。

### 2. 主题切换限制

MaterialDesignThemes的某些颜色可能需要应用重启才能完全生效：
- 某些资源字典缓存
- 部分静态资源引用

**解决方案**：99%的场景下都能立即生效，极少数情况需要重启应用。

### 3. 性能考虑

对所有窗口调用UpdateLayout()可能有轻微性能影响：
- 窗口数量越多，刷新耗时越长
- 复杂布局的刷新成本更高

**影响评估**：
- 单次切换耗时：<100ms（通常情况）
- 用户感知：无感知（切换不频繁）
- 优化方案：已使用try-catch保护，单个窗口失败不影响其他
