## 2.1 MainWindow 设计

### 目标
设计单窗体容器，无边框自定义标题栏，支持多视图切换和全屏显示。

### 设计要点

| 特性 | 要求 |
|-----|------|
| 窗口样式 | 无边框 (WindowStyle="None") |
| 标题栏 | 自定义，包含Logo、标题、版本号、窗口按钮 |
| 最小尺寸 | 1280 x 720 |
| 默认尺寸 | 1920 x 1080 |
| 内容区域 | ContentControl（用于视图切换） |

### 自定义标题栏内容

| 区域 | 内容 |
|-----|------|
| 左侧 | Logo图片（无单位标识） |
| 中间 | 应用名称 |
| 右侧-版本 | 版本号（点击可查看完整版本） |
| 右侧-按钮 | 最小化、全屏/还原、关闭 |

### 全屏功能

| 操作 | 说明 |
|-----|------|
| 进入全屏 | F11键 或 点击全屏按钮 |
| 退出全屏 | ESC键 或 点击还原按钮 |
| 图标切换 | 全屏时显示还原图标，正常时显示最大化图标 |

### 关闭窗口

- 点击关闭按钮时弹出二次确认对话框
- 确认信息："确定要退出系统吗？"
- 提供"是"和"否"按钮

### 窗口拖拽

- 标题栏区域支持鼠标拖拽移动窗口

### MainWindowViewModel 职责

| 功能 | 说明 |
|-----|------|
| WindowState | 窗口状态属性 |
| MinimizeCommand | 最小化命令 |
| ToggleFullScreenCommand | 全屏切换命令 |
| ExitFullScreenCommand | 退出全屏命令 |
| CloseCommand | 关闭命令（带确认） |
| ShowVersionCommand | 显示版本信息命令 |
| CurrentContent | 当前显示的内容（视图） |
| NavigateTo | 导航到指定视图的方法 |

## 2.2 登录界面 (LoginView)

### 目标
实现三级权限用户登录功能，遵循 DevelopmentSpecifications.md 规范。

### UI设计

| 元素 | 说明 |
|-----|------|
| 布局 | 居中显示的 Material Design 卡片 |
| Logo | 顶部显示应用Logo |
| 标题 | 应用名称（中英文） |
| 账号输入 | TextBox，最大长度20 |
| 密码输入 | PasswordBox，最大长度20，支持显隐切换 |
| 记住密码 | CheckBox |
| 错误提示 | 红色文字显示 |
| 登录按钮 | 主按钮，支持加载状态 |
| 游客登录 | 文字链接按钮 |
| 版本号 | 右下角显示 |

### 登录验证规则（参考 DevelopmentSpecifications.md）

| 规则 | 说明 |
|-----|------|
| 账号长度 | 3-20字符 |
| 密码长度 | 6-20字符 |
| 空格处理 | 前后有空格视为错误 |
| 错误提示 | 统一显示"账号或密码不正确"（避免猜测） |
| 禁用账号 | 显示"该账户已被禁用" |
| 失败限制 | 无次数限制 |

### 密码显隐切换

- 默认隐藏密码
- 点击眼睛图标切换显示/隐藏
- 图标：Eye / EyeOff

### 记住密码功能

- 勾选后，账号和密码保存到配置文件
- 密码需加密存储
- 下次启动自动填充

### 游客登录

- 无需输入账号密码
- 直接以游客身份进入
- 游客不进入患者选择界面，直接进入主界面
- 游客数据退出后自动清除

### LoginViewModel 职责

| 属性/命令 | 说明 |
|----------|------|
| Username | 账号 |
| Password | 密码 |
| RememberPassword | 记住密码 |
| IsPasswordHidden | 密码是否隐藏 |
| ErrorMessage | 错误信息 |
| IsLoggingIn | 登录中状态 |
| CanLogin | 是否可登录 |
| LoginCommand | 登录命令 |
| GuestLoginCommand | 游客登录命令 |
| TogglePasswordVisibilityCommand | 密码显隐切换 |

### 登录流程

```
输入账号密码 → 验证格式 → 调用认证服务 → 检查账号状态
                                              ↓
                            成功 → 保存会话 → 保存记住密码 → 导航到患者选择
                            失败 → 显示错误信息
```

### 删除权限判断逻辑

```csharp
private bool CanDeletePatient(Patient patient)
{
    if (patient == null) return false;

    var currentUser = _sessionService.CurrentUser;
    if (currentUser == null) return false;

    // 管理员可删除任意患者
    if (currentUser.Role == UserRole.Administrator) return true;

    // 操作员只能删除自己创建的患者
    if (currentUser.Role == UserRole.Operator)
        return patient.CreatedBy == currentUser.Id;

    return false;
}
```

## 2.6 视图导航实现

### 导航流程设计

```
App启动 → 显示MainWindow → 导航到LoginView
登录成功 → MainWindow导航到PatientSelectionView
选择患者 → MainWindow导航到MainContainerView
切换菜单 → MainContainerView内部切换子视图
退出登录 → MainWindow导航回LoginView
```

### 视图切换实现

使用自定义的`NavigationService`实现视图导航：

1. **MainWindow**中的`ContentControl`绑定到`CurrentContent`属性
2. `NavigationService`提供`NavigateTo`方法来切换视图
3. 每个视图都对应一个ViewModel，通过DataTemplate关联

### 数据传递

- 登录成功后的用户信息通过服务层传递
- 选择的患者信息通过服务层传递
- 视图间的数据共享通过服务层实现，避免直接传递
