## 问题描述

程序启动时弹出错误弹窗，显示：
```
数据库初始化失败：Must add values for the following parameters
```

## 问题原因汇总

在 `DatabaseInitializer.cs` 的 `SeedDataAsync` 方法中，存在以下问题：

### 问题 1：混合使用字面值和参数（已修复）

### 问题 2：用户表缺少参数（已修复）

### 问题 3：参数重复使用问题 ?? 最新发现

**错误代码**：
```csharp
// ? 错误：同一个参数 @Now 在 SQL 中重复使用
await db.ExecuteNonQueryAsync(@"
    INSERT OR IGNORE INTO Departments (Name, Phone, CreatedAt, UpdatedAt)
    VALUES (@Name, @Phone, @Now, @Now);
", new 
{ 
    Name = "默认科室", 
    Phone = "",
    Now = now  // 参数只提供一次，但 SQL 中用了两次
});
```

**根本原因**：
- ToolHelper.Database 的 `AddParameters` 方法从参数对象创建 DbParameter
- Microsoft.Data.Sqlite 对参数的处理方式：**每个占位符需要独立的参数**
- 当 SQL 中 `@Now` 出现两次时，需要两个独立的参数值

**修复后**：
```csharp
// ? 正确：每个占位符使用独立的参数名
await db.ExecuteNonQueryAsync(@"
    INSERT OR IGNORE INTO Departments (Name, Phone, CreatedAt, UpdatedAt)
    VALUES (@Name, @Phone, @CreatedAt, @UpdatedAt);
", new 
{ 
    Name = "默认科室", 
    Phone = "",
    CreatedAt = now,
    UpdatedAt = now  // 值相同，但参数名不同
});
```

### 问题 4：多行 INSERT 参数过多（已修复）

**错误代码**：
```csharp
// ? 错误：一个 SQL 中有太多参数，容易出错
await db.ExecuteNonQueryAsync(@"
    INSERT OR IGNORE INTO SystemSettings (Key, Value, ValueType, UpdatedAt)
    VALUES 
        (@Key1, @Value1, @Type1, @Now),
        (@Key2, @Value2, @Type2, @Now),
        ...
", new { Key1 = ..., Key2 = ..., ... });
```

**修复后**：
```csharp
// ? 正确：使用循环单独执行每条 INSERT
var settings = new[]
{
    ("UnitName", "", "string"),
    ("LogoPath", "", "string"),
    ...
};

foreach (var (key, value, type) in settings)
{
    await db.ExecuteNonQueryAsync(@"
        INSERT OR IGNORE INTO SystemSettings (Key, Value, ValueType, UpdatedAt)
        VALUES (@Key, @Value, @ValueType, @UpdatedAt);
    ", new { Key = key, Value = value, ValueType = type, UpdatedAt = now });
}
```

## 核心规则总结

### ?? ToolHelper.Database + Microsoft.Data.Sqlite 的铁律

1. **每个占位符必须有唯一的参数名**
   ```csharp
   // ? 错误：@Now 重复使用
   VALUES (@Name, @Now, @Now)

   // ? 正确：每个位置独立命名
   VALUES (@Name, @CreatedAt, @UpdatedAt)
   ```

2. **不能使用任何字面值**
   ```csharp
   // ? 错误
   VALUES ('literal', 123, @Param)

   // ? 正确
   VALUES (@Value1, @Value2, @Param)
   ```

3. **复杂多行 INSERT 拆分执行**
   ```csharp
   // ? 错误：参数太多容易出错
   VALUES (@A1, @B1), (@A2, @B2), (@A3, @B3)...

   // ? 正确：循环单独执行
   foreach (var item in items)
   {
       await db.ExecuteNonQueryAsync("INSERT ... VALUES (@A, @B)", 
           new { A = item.A, B = item.B });
   }
   ```

4. **参数对象属性必须与占位符一一对应**
   ```csharp
   // SQL 中有: @Name, @Phone, @CreatedAt, @UpdatedAt
   // 参数对象必须有: Name, Phone, CreatedAt, UpdatedAt
   ```

## 经验教训

### ? 最佳实践

1. **完全参数化**：所有 SQL 值都应通过参数传递，包括空字符串
   ```csharp
   // 正确
   VALUES (@Name, @Phone, @CreatedAt)
   // 错误
   VALUES ('Name', '', @CreatedAt)
   ```

2. **参数对象匹配**：确保匿名对象的属性名与 SQL 中的占位符一致
   ```csharp
   VALUES (@Name, @Phone)  →  new { Name = "...", Phone = "..." }
   ```

3. **多行 INSERT 参数化**：每行都需要独立的参数
   ```csharp
   VALUES (@Key1, @Value1), (@Key2, @Value2)
   ```

### ? 避免的错误模式

1. **混合字面值和参数**
   ```csharp
   // 错误
   VALUES ('literal', @Parameter)
   ```

2. **参数不完整**
   ```csharp
   // 错误：SQL 有 3 个参数，但只提供了 1 个
   VALUES (@A, @B, @C), new { A = "value" }
   ```

3. **参数名不匹配**
   ```csharp
   // 错误
   VALUES (@Name), new { UserName = "value" }
   ```

## 防止重犯的检查清单

在编写数据库操作代码时，**每次都要检查**：

### ? SQL 语句检查
- [ ] SQL 中没有任何字面值（数字、字符串、布尔值等）
- [ ] 所有值都使用 `@ParameterName` 格式
- [ ] 参数名称清晰易懂

### ? 参数对象检查
- [ ] 参数对象包含所有占位符对应的属性
- [ ] 属性名与占位符名称完全匹配（区分大小写）
- [ ] 没有遗漏任何参数

### ? 特殊值检查
- [ ] 数字值（如 `1`, `0`）已参数化
- [ ] 布尔值已转换为整数并参数化
- [ ] 空字符串 `""` 已参数化
- [ ] NULL 值正确处理

### ? 多行 INSERT 检查
- [ ] 每一行的每个值都有对应参数
- [ ] 参数命名有规律（如 Key1, Key2, ...）
- [ ] 参数数量正确

## 调试技巧

当遇到 "Must add values for the following parameters" 错误时：

1. **找到错误的 SQL 语句**
   ```csharp
   // 在 catch 块中打印 SQL
   catch (Exception ex)
   {
       Console.WriteLine($"SQL: {sqlStatement}");
       Console.WriteLine($"Error: {ex.Message}");
   }
   ```

2. **检查参数占位符和参数对象**
   - 数一数 SQL 中有多少个 `@` 符号
   - 数一数参数对象中有多少个属性
   - 确保数量和名称一致

3. **使用工具辅助检查**
   ```csharp
   // 打印参数对象
   var paramObj = new { Name = "test", Phone = "" };
   var properties = paramObj.GetType().GetProperties();
   foreach (var prop in properties)
   {
       Console.WriteLine($"{prop.Name} = {prop.GetValue(paramObj)}");
   }
   ```
