## 一、现状分析与冲突识别

### 1.1 现有代码现状

#### 1.1.1 MeasurementViewModel.cs（需完全重写）

当前实现是简单的"设备连接 + 录制"模式：

```
现有功能：
├── 设备列表（硬编码模拟设备）
├── 连接/断开命令
├── 开始/停止录制命令
├── 实时数据模拟（步频、速度）
└── 无步骤向导逻辑
```

**结论**：与设计方案的四步向导模式完全不同，需要**完全重写**。

#### 1.1.2 MeasurementView.xaml（需完全重写）

当前是简单的两列布局：
- 左侧：视频预览占位（黑色区域）
- 右侧：设备选择、连接按钮、录制按钮、实时数据

**结论**：与设计方案的三列+底部抽屉布局完全不同，需要**完全重写**。

#### 1.1.3 MeasurementRecord 模型（需扩展）

现有字段：
```csharp
public class MeasurementRecord
{
    public int Id { get; set; }
    public int PatientId { get; set; }
    public int OperatorId { get; set; }
    public DateTime MeasurementDate { get; set; }
    public MeasurementStatus Status { get; set; }
    public string? VideoFilePath { get; set; }      // ⚠️ 单一视频路径
    public int? DurationSeconds { get; set; }
    public bool IsGuestData { get; set; }
    public string? Remark { get; set; }
    // ...
}
```

**缺失字段**（需新增）：
- `FrontVideoPath` - 正面视频路径
- `SideVideoPath` - 侧面视频路径
- `MeasurementType` - 测量类型（自然步行/快走/慢走/其他）
- `MeasurementName` - 测量名称
- `WalkwayLength` - 步道长度
- `VideoSpec` - 视频规格
- `AnalysisStage` - 分析阶段
- `KeypointsCompleted` - 关键点分析完成标记
- `EventsCompleted` - 事件分析完成标记
- `KinematicsCompleted` - 运动学分析完成标记

#### 1.1.4 枚举定义（需扩展）

现有枚举 `MeasurementStatus`：
```csharp
public enum MeasurementStatus
{
    Pending = 0,      // 待处理
    InProgress = 1,   // 进行中
    Completed = 2,    // 已完成
    Cancelled = 3,    // 已取消
    Failed = 4        // 分析失败
}
```

**需新增枚举**：
- `MeasurementType` - 测量类型
- `AnalysisStage` - 分析阶段
- `VideoSpec` - 视频规格
- `ImportStrategy` - 导入策略

#### 1.1.5 多语言资源（需扩展）

现有结构：
- `Resources/Localization/Strings.zh.xaml`
- `Resources/Localization/Strings.en.xaml`

**适配策略**：
- 继续使用现有文件（不改文件名）
- 在现有文件中添加 `MA.*` 前缀的资源键
- 复用已有的通用键（如 `Next`、`Back`、`Cancel` 等）

#### 1.1.6 对话框服务（可复用）

现有方式：
```csharp
await DialogHost.Show(new ConfirmDialog { ... }, "RootDialog");
```

**适配策略**：
- 继续使用 `DialogHost.Show()` 方式
- 不需要新建 `IDialogService` 接口
- 新增特定对话框组件（如视频不完整确认对话框）

#### 1.1.7 导航服务（可复用）

现有方式：
- `MainContainerViewModel` 通过 DI 获取 View 并设置 `CurrentContent`
- 四步向导内部使用 `ContentControl + DataTemplate` 切换

**适配策略**：
- 外层导航保持不变
- 向导内部步骤切换由 `MeasurementViewModel` 自行管理

### 1.2 冲突汇总与解决方案

| 冲突项 | 现状 | 设计方案要求 | 解决方案 |
|-------|------|-------------|---------|
| **ViewModel** | 简单录制模式 | 四步向导模式 | 完全重写 |
| **View** | 两列布局 | 三列+底部抽屉 | 完全重写 |
| **视频路径** | 单一 `VideoFilePath` | 双路径 Front/Side | 扩展模型字段 |
| **测量类型** | 无 | 自然步行/快走/慢走/其他 | 新增枚举和字段 |
| **分析阶段** | 无 | Keypoints/Events/Kinematics | 新增枚举和字段 |
| **资源文件名** | `Strings.zh.xaml` | `StringResources.zh-CN.xaml` | 保持现有命名 |
| **资源键前缀** | 无前缀 | `MA.*` 前缀 | 新增带前缀的键 |
| **对话服务** | `DialogHost.Show()` | `IDialogService` 接口 | 保持现有方式 |

### 3.3 数据库迁移策略

由于需要新增多个字段，需要更新 `DatabaseInitializer.cs`：

1. **版本号递增**：数据库版本从当前值 +1
2. **字段迁移**：使用 SqlSugar 的 `CodeFirst` 自动同步表结构
3. **默认值处理**：新字段使用默认值，不影响现有数据
4. **VideoFilePath 处理**：保留现有字段，新增 `FrontVideoPath`/`SideVideoPath` 不删除旧字段

#### 5.2 分析模拟逻辑

```csharp
[RelayCommand(CanExecute = nameof(CanStartAnalyze))]
private async Task StartAnalyzeAsync()
{
    IsAnalyzing = true;
    IsTaskDrawerOpen = true;
    
    // 模拟分析进度
    for (int i = 0; i <= 100; i += 10)
    {
        AnalysisProgress = i;
        await Task.Delay(300);
        
        if (!IsAnalyzing) break; // 取消检测
    }
    
    if (IsAnalyzing)
    {
        // 完成
        CompletedStage = SelectedStage;
        UpdateStageCompletionFlags();
    }
    
    IsAnalyzing = false;
}

private bool CanStartAnalyze() => HasFrontVideo && HasSideVideo && !IsAnalyzing;
```
