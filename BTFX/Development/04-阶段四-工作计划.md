## 六、风险与注意事项

### 6.1 已知风险

1. **数据库版本升级**
   - 风险：升级脚本可能导致数据丢失
   - 措施：升级前自动备份

2. **并发访问**
   - 风险：SQLite 并发写入限制
   - 措施：使用 WAL 模式，控制写入频率

3. **大文件导入**
   - 风险：内存溢出
   - 措施：使用流式读取

### 6.2 注意事项

1. **工具类使用**：
   - SqliteHelper 需要使用 `using` 语句或正确 Dispose
   - JsonHelper 可以复用实例
   - ExcelHelper/CsvHelper 是泛型类，需要指定类型

2. **事务处理**：
   - 多表操作必须使用事务
   - 事务内不要有耗时操作

3. **日期时间**：
   - SQLite 存储为 TEXT，格式：`yyyy-MM-dd HH:mm:ss`
   - 查询时需要正确转换

## 九、已知问题与修复历史

### 问题 1：数据库初始化参数错误 ? 已修复

**发现时间**: 2025-01  
**修复次数**: 3 次

**问题描述**: SQL INSERT 语句参数处理问题，导致 "Must add values for the following parameters" 错误

**修复内容**:
1. 第一次修复：Departments 表和 SystemSettings 表的字面值
2. 第二次修复：Users 表的 `IsEnabled` 和 `IsBuiltIn` 字面值
3. 第三次修复：**参数重复使用问题** - `@Now` 在 SQL 中出现两次，需要使用独立参数名

**详细文档**: [数据库初始化参数错误修复记录](./数据库初始化参数错误修复记录.md)

**核心经验**:
- ToolHelper.Database + Microsoft.Data.Sqlite **不支持参数名重复使用**
- 每个 SQL 占位符必须有**唯一的参数名**
- 即使值相同，也要用不同的参数名（如 `@CreatedAt`, `@UpdatedAt`）
- 复杂多行 INSERT 应拆分为循环单独执行
