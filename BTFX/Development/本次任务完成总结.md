**问题描述**：

- 缺少对 admin 账户的特殊处理

**修复内容**：

#### 2.1 密码编码/解码修复
```csharp
// 保存：Base64 编码
var passwordBytes = System.Text.Encoding.UTF8.GetBytes(Password.Trim());
settings.Credentials.PasswordHash = Convert.ToBase64String(passwordBytes);

// 加载：Base64 解码
var passwordBytes = Convert.FromBase64String(settings.Credentials.PasswordHash);
Password = System.Text.Encoding.UTF8.GetString(passwordBytes);
```

#### 2.2 Admin 账户保护
```csharp
// 加载时跳过 admin 账户
if (settings.Credentials.Username.Equals("admin", StringComparison.OrdinalIgnoreCase))
{
    return;
}

// 保存时不保存 admin 账户密码
if (RememberPassword && !Username.Trim().Equals("admin", StringComparison.OrdinalIgnoreCase))
{
    // 保存密码
}
```

### ✅ 4. 登录按钮问题修复

**问题描述**：
完成记住密码修复后，登录按钮无法点击

**根本原因**：
使用 `[RelayCommand(CanExecute = nameof(CanLogin))]` 时，属性改变没有通知命令重新评估 CanExecute

**修复方案**：
添加 `[NotifyCanExecuteChangedFor]` 特性

```csharp
[ObservableProperty]
[NotifyCanExecuteChangedFor(nameof(LoginCommand))]
private string _username = string.Empty;

[ObservableProperty]
[NotifyCanExecuteChangedFor(nameof(LoginCommand))]
private string _password = string.Empty;

[ObservableProperty]
[NotifyCanExecuteChangedFor(nameof(LoginCommand))]
private bool _isLoggingIn;
```

**技术原理**：
- `[ObservableProperty]` 只触发 PropertyChanged 事件
- `[NotifyCanExecuteChangedFor]` 额外通知命令重新评估 CanExecute
- 这是 CommunityToolkit.Mvvm 的标准用法

**修改文件**：
- `BTFX/ViewModels/LoginViewModel.cs`
