## 3.1 测量界面 (MeasurementView) - 暂留框架

### 说明

> ?? **重要**: 测量模块的核心算法功能尚未完成，本阶段仅搭建界面框架和基础交互，具体功能待算法完成后实现。

### 系统功能概述

BTFX是步态智能分析系统，测量模块的主要功能：
1. 连接摄像机采集患者行走视频
2. 对视频进行关节点识别
3. 分析步态参数
4. 输出分析结果

### 界面框架设计

| 区域 | 内容 | 说明 |
|-----|------|------|
| 左上 | 视频预览区 | 显示摄像机实时画面/录制视频 |
| 右上 | 控制面板 | 设备连接、录制控制 |
| 左下 | 参数显示区 | 步态参数实时/结果显示 |
| 右下 | 操作按钮区 | 开始测量、停止、保存、取消 |

### 控制面板内容

| 控件 | 说明 |
|-----|------|
| 设备选择 | 下拉选择摄像机设备 |
| 连接按钮 | 连接/断开设备 |
| 状态指示 | 设备连接状态 |
| 分辨率选择 | 视频分辨率（待定） |
| 帧率选择 | 视频帧率（待定） |

### 操作流程（预期）

```
选择设备 → 连接设备 → 开始录制 → 患者行走 → 停止录制 → 自动分析 → 显示结果 → 保存数据
```

### 游客模式特殊处理

- 可以执行测量
- 测量数据保存到临时文件夹
- 退出登录时自动删除临时数据

## 3.3 报告界面 (ReportView)

### 目标
基于测量数据生成报告，支持预览、打印、导出。

### UI布局

| 区域 | 内容 |
|-----|------|
| 左侧 | 数据选择/报告列表 |
| 右侧 | 报告预览区 |
| 底部 | 操作按钮 |

### 功能模式

| 模式 | 说明 |
|-----|------|
| 生成报告 | 选择测量数据生成新报告 |
| 重新生成 | 对已有报告重新生成，覆盖原报告 |
| 报告管理 | 查看/编辑/打印历史报告 |

### 报告生成流程

```
选择测量数据 → 检查是否已有报告 → [有] 提示是否覆盖 → 选择报告模板 → 填写/编辑医生意见 → 预览报告 → 保存/打印
                                   → [无] 直接进入生成流程
```

### 报告覆盖逻辑

- 同一测量数据可以重新生成报告
- 重新生成时覆盖原有报告
- 需要确认提示："该测量数据已有报告，是否覆盖？"
- 保留报告编号，更新内容和时间戳

### 报告内容（参考 DevelopmentSpecifications.md）

| 内容 | 说明 |
|-----|------|
| 医院/单位名称 | 系统设置中配置 |

| 测量信息 | 日期、操作员 |
| 步态参数数据 | 各项分析结果（待定） |
| 图表/图片 | 步态分析图表（待定） |
| 医生意见 | **可编辑字段** |
| 评分标准 | 评估报告需附上 |
| 页码 | 格式：第 X 页 / 共 Y 页 |

## 待确认问题（已全部确认）

> **Q-3.1**: 测量界面的视频预览控件使用什么？ - **使用支持快进/慢放/倍速调节的第三方库**
>
> **Q-3.2**: 步态参数具体包括哪些指标？需要显示哪些参数？ - **步幅、步频、步速、单脚支撑时间、双脚支撑时间等，具体需与算法团队确认**
>
> **Q-3.3**: 报告模板是否需要支持自定义？还是使用固定模板？ - **目前使用固定模板，后续版本可考虑自定义**
>
> **Q-3.4**: 报告中的图表使用什么库生成？需要确认图表类型。 - **使用OxyPlot库，主要为折线图和柱状图**
>
> **Q-3.5**: 自动备份的默认设置：频率、时间、保留数量？ - **每天凌晨2点备份，保留最近7个备份文件**
>
> **Q-3.6**: 是否需要支持数据库配置（切换数据库文件位置）？ - **是的，支持配置数据库文件位置，修改后需重启**
>
> **Q-3.7**: 日志保留策略：保留多少天？文件大小限制？ - **保留30天，单个文件限制10MB，超过后创建新文件**
